# v2.6 審計日誌與舉報

## 目標

記錄所有重要操作，提供可查詢的活動日誌。

## 功能

### 1. 審計日誌
- 記錄所有操作：登入、上傳、下載、刪除、分享、權限變更
- 欄位：時間、使用者、動作、檔案路徑、IP、UA
- 可篩選：依使用者、依動作類型、依時間範圍
- 分頁查詢

### 2. 事件通知
- 重要操作即時通知（可選）
- 渠道：Email / Webhook
- 觸發條件：大量下載、異常登入、新成員加入

### 3. 匯出功能
- 日誌匯出 CSV/JSON
- 可設定時間範圍

## 資料庫

```sql
CREATE TABLE audit_log (
    id INTEGER PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    action TEXT,
    resource_type TEXT,
    resource_id TEXT,
    file_path TEXT,
    ip TEXT,
    user_agent TEXT,
    metadata TEXT,
    created_at TEXT
);

CREATE TABLE notifications (
    id INTEGER PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    type TEXT,
    title TEXT,
    message TEXT,
    read INTEGER DEFAULT 0,
    created_at TEXT
);

CREATE TABLE webhooks (
    id INTEGER PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    url TEXT,
    events TEXT,
    secret TEXT,
    active INTEGER DEFAULT 1,
    created_at TEXT
);
```

## API 端點

| Method | Endpoint | 說明 |
|--------|----------|------|
| GET | `/api/audit` | 審計日誌列表 |
| GET | `/api/audit/export` | 匯出日誌 |
| GET | `/api/notifications` | 通知列表 |
| PUT | `/api/notifications/{id}/read` | 標記已讀 |
| POST | `/api/webhooks` | 建立 Webhook |
| GET | `/api/webhooks` | Webhook 列表 |
| DELETE | `/api/webhooks/{id}` | 刪除 Webhook |
