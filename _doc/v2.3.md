# v2.3 監控與可靠性

## 目標

建立健康檢查、備份系統、容器自動化管理。

## 功能

### 1. 健康檢查
- `/health` — 主 server 狀態（JSON）
- `/health/docker` — Docker daemon 狀態
- `/health/containers` — 所有容器狀態
- 供 Load Balancer / 外部監控使用

### 2. 備份系統
- 每日凌晨 3 點自動備份
- 備份內容：SQLite metadata DB + 使用者上傳檔案
- 備份儲存：`backups/` 目錄，保留 7 天
- 備份腳本：`k8s/backup.sh`

### 3. 容器自動重啟
- 監控容器狀態（每分鐘檢查）
- Crash → 自動重啟
- 記錄 restart 次數
- 3 次 restart 失敗 → 寄送警告

### 4. 使用量統計
- 每日統計：總上傳量、總下載量、活躍使用者數
- 儲存到 `stats/` 目錄（JSON）
- 可查詢歷史統計

### 5. 斷線重連
- 使用者斷線後自動重連 container
- WebSocket heartbeat 機制

## 健康檢查回應格式

```json
{
  "status": "healthy",
  "docker": "connected",
  "containers": {
    "total": 10,
    "running": 8,
    "exited": 2
  },
  "timestamp": "2026-05-12T03:00:00Z"
}
```

## 備份腳本

```bash
#!/bin/bash
# k8s/backup.sh
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="backups"
mkdir -p $BACKUP_DIR

# 備份資料庫
cp uploads/*.db "$BACKUP_DIR/box5_$DATE.db"

# 打包上傳檔案
tar -czf "$BACKUP_DIR/uploads_$DATE.tar.gz" uploads/

# 清理 7 天前備份
find $BACKUP_DIR -mtime +7 -delete
```

## Cron 設定

```bash
# /etc/crontab
0 3 * * * root /path/to/backup.sh   # 每日凌晨 3 點備份
* * * * * root /path/to/check_containers.sh  # 每分鐘檢查容器
```
