# v1.3 容器埠號修復 + 內建 vim/code5

## 問題

執行 k8s 版本時，登入後 `/` 頁面可正常顯示檔案列表，但點擊 **View** 或 **Editor** 時畫面一直載入（轉圈圈）沒有回應。

## 根因

`get_user_port()` 的實作邏輯為：
1. 檢查 `.port` 檔案 → 不存在（未寫入）
2. 查詢 Docker API 取得容器 port mapping → 可能因 Docker API 回應慢而拋出例外
3. 回退到 hash 計算埠號：`20000 + sum(ord(c) for c in username) % 10000`

ccc 使用者的 hash 埠號為 20297，但 Docker 實際分配的埠號為 51181（測試環境）或 55414（測試環境），兩者不符。因此 view/editor 在呼叫容器 API 時連線逾時，導致畫面卡住。

## 修復

### 1. 寫入 `.port` 檔案（k8s/main.py）

在 `create_user_container()` 中，取得 Docker 分配的實際埠號後，寫入使用者目錄下的 `.port` 檔案：

```python
port_file = os.path.join(user_dir, ".port")
with open(port_file, "w") as f:
    f.write(str(user_port))
```

讓後續 `get_user_port()` 可直接從 `.port` 檔案讀取埠號，不需經由 Docker API 查詢，也不受 hash 回退機制的影響。

### 2. 容器映像檔內建 vim（k8s/Dockerfile.box5）

apt-get install 加入 `vim`。

### 3. 容器映像檔內建 code5（k8s/Dockerfile.box5）

```dockerfile
RUN uv pip install --system code5
```

在容器建立時從 host 環境變數 `NVIDIA_API_KEY` 傳入（方案 A，不進 git）：

```python
nvidia_key = os.environ.get("NVIDIA_API_KEY", "")
if nvidia_key:
    env_vars.append(f"NVIDIA_API_KEY={nvidia_key}")
```

### 4. 測試修復（k8s/tests/test_main.py）

`TestDirectoryCreation::test_create_container_creates_directories` 移除過度寬泛的 `@patch('main.os.makedirs')`，讓目錄建立正常執行，避免 `shutil.copytree` 因目標目錄不存在而失敗。
