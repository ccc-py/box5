# box5 v1.0

## New Features & Fixes
1. **Static Website Rendering Architecture (Path-based Routing)**
   - Overhauled the previous UI viewer logic (`/view/{id}`) to natively support full path routing paths (`/view/public/ccc.md`).
   - Relative URL linking (such as `./img/ccc.jpg` inside `.md` and `.html` files) now automatically resolves flawlessly without requiring `<base href...>` tag HTML injection.
   - Replaced custom raw data handling with native `Response` streaming — static assets (images, html, css, js) now display automatically with native content headers in browsers, identical to how standard static web servers respond.
2. **Path-based API Retrieval Methods**
   - Implemented `get_file_by_path` (`/files/bypath/{path}`) and `get_public_file_by_path` API querying to instantly extract latest document IDs and directories from structural matching, allowing arbitrary web traversal. 
3. **Database Version Deduplication**
   - Solved a database glitch where `sql5` `MAX(id)` grouped subqueries failed to eliminate historical file iterations on UI screens.
   - Forced strict backend level python iteration filtering (comparing array tuples), fixing duplicate histories appearing dynamically in both Private and Public files tabs.
4. **UI Navigation Bug Fixes**
   - Corrected Jinja2 `{% else %}` routing flaw which previously prevented `subfolders` from continuing to render visually while browsing within an active `current_folder`. Subdirectories and `../ (Up)` navigators operate cleanly.
