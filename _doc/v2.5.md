# v2.5 組織與協作

## 目標

支援多人協作，建立組織、共享資料夾、成員管理。

## 功能

### 1. 組織
- 建立組織（名稱、圖示）
- 邀請成員（email 邀請連結）
- 成員角色：Owner / Admin / Member
- 組織設定：成員配額上限

### 2. 共享資料夾
- 在組織下建立共享資料夾
- 資料夾成員權限：讀取 / 讀寫
- 組織成員自動可見共享資料夾
- 個別資料夾可設定更細緻權限

### 3. 組織管理
- 成員列表（邀請/移除）
- 角色變更
- 活動日誌（成員操作）

## 資料庫 Schema

```sql
CREATE TABLE organizations (
    id INTEGER PRIMARY KEY,
    name TEXT,
    owner_id INTEGER REFERENCES users(id),
    created_at TEXT
);

CREATE TABLE org_members (
    id INTEGER PRIMARY KEY,
    org_id INTEGER REFERENCES organizations(id),
    user_id INTEGER REFERENCES users(id),
    role TEXT DEFAULT 'member',
    joined_at TEXT,
    UNIQUE(org_id, user_id)
);

CREATE TABLE shared_folders (
    id INTEGER PRIMARY KEY,
    org_id INTEGER REFERENCES organizations(id),
    name TEXT,
    path TEXT,
    owner_id INTEGER REFERENCES users(id),
    created_at TEXT
);

CREATE TABLE folder_permissions (
    id INTEGER PRIMARY KEY,
    folder_id INTEGER REFERENCES shared_folders(id),
    user_id INTEGER REFERENCES users(id),
    permission TEXT DEFAULT 'read'
);
```

## API 端點

| Method | Endpoint | 說明 |
|--------|----------|------|
| POST | `/api/orgs` | 建立組織 |
| GET | `/api/orgs` | 我的組織列表 |
| POST | `/api/orgs/{id}/invite` | 邀請成員 |
| POST | `/api/orgs/{id}/join` | 加入組織（用邀請連結） |
| GET | `/api/orgs/{id}/members` | 成員列表 |
| PUT | `/api/orgs/{id}/members/{user_id}` | 變更角色 |
| DELETE | `/api/orgs/{id}/members/{user_id}` | 移除成員 |
| GET | `/api/orgs/{id}/folders` | 共享資料夾 |
| POST | `/api/orgs/{id}/folders` | 建立共享資料夾 |
| PUT | `/api/folders/{id}/permissions` | 設定資料夾權限 |

## 前端頁面

- `/orgs` — 組織列表
- `/orgs/{id}` — 組織首頁（成員、資料夾）
- `/orgs/{id}/settings` — 組織設定
