# v2.4 三種模式：簡單同步、多用戶（無容器）、多用戶（Docker 隔離）

## 目標

把 Box5 從兩種模式擴展為三種模式，滿足不同使用情境：

1. **簡單模式** — 沒有 Web 介面，只有本機資料夾 ↔ Server 資料夾的雙向同步
2. **多用戶模式（無容器）** — 多使用者共享一個 Server，資料夾隔離，無 Docker
3. **多用戶模式（Docker 隔離）** — 現有的完整隔離模式

## 三種模式比較

| 特色 | 簡單模式 | 多用戶（無容器） | 多用戶（Docker 隔離） |
|------|----------|------------------|------------------------|
| Web 介面 | 無 | 有 | 有 |
| 多使用者 | 否（單一資料夾） | 是 | 是 |
| 容器隔離 | 無 | 無 | 是 |
| SSH 存取 | 無 | 無 | 是 |
| 設定難度 | 最簡單 | 中等 | 最複雜 |
| 適合場景 | 個人同步 | 小團隊、內網 | 生產環境、多租用戶 |

## 模式一：簡單模式（Simple Sync）

### 概念

沒有 Web 介面，不需要帳號系統。只要啟動，指定的本機資料夾就會自動和 Server 的指定資料夾雙向同步。

### 運作方式

```
本機 ./sync/   ↔  Server (用一個指定的資料夾存放所有檔案)
```

不需要 Docker，不需要帳號，不需要 Web 介面。只要：
1. Server 啟動
2. Client 啟動並指定本機資料夾和 Server URL
3. 自動雙向同步

### Server 端

```bash
cd k8s
BOX5_MODE=simple BOX5_ROOT=/data/box5 python -m uvicorn main:app --host 0.0.0.0 --port 8000
```

只需要設定一個資料夾作為根目錄，所有檔案都存在這裡。

### Client 端

```bash
cd k8s
python simple_sync.py \
  --server http://localhost:8000 \
  --local-folder ./sync \
  --remote-folder ""      # Server 的根目錄
```

或環境變數：

```bash
export BOX5_SERVER=http://localhost:8000
export BOX5_LOCAL_FOLDER=./sync
export BOX5_REMOTE_FOLDER=   # 空字串 = Server 根目錄
python simple_sync.py
```

### 實作

```python
# simple_sync.py（新檔案）
# - 登入或匿名同步
# - 啟動後持續監控本機資料夾
# - 定時同步：上傳新的、刪除伺服器上被移除的
# - 差量同步：比對檔案 hash，只同步有變化的
# - 衝突處理：保留兩版本（filename.ext vs filename.ext.1）
```

### 簡單模式功能

- 雙向同步（Watch 模式）
- 支援 .gitignore 風格的排除規則
- 差量同步（比對 hash）
- 衝突處理（保留雙方版本）
- 離線支援（重啟後自動補上）
- 進度顯示

## 模式二：多用戶模式（無容器）

### 概念

一個 Server，多個使用者。每個使用者的資料存在 `<root>/<username>/` 資料夾下，沒有 Docker 容器，完全依賴資料夾隔離。

### 運作方式

```
root/
├── alice/
│   ├── sync/          ← alice 的檔案
│   └── public/         ← alice 的公開檔案
├── bob/
│   ├── sync/          ← bob 的檔案
│   └── public/         ← bob 的公開檔案
└── charlie/
    └── ...
```

每個使用者在 Server 上有一個以自己帳號命名的資料夾。只能存取自己的資料夾。

### Server 端

```bash
cd k8s
BOX5_MODE=multi        # 多用戶模式
BOX5_ROOT=/data/box5   # 所有使用者資料的根目錄
BOX5_NO_DOCKER=1       # 不使用 Docker
python -m uvicorn main:app --host 0.0.0.0 --port 8000
```

### Web 介面（維持現有功能）

- 登入 / 註冊
- 檔案列表、下載、上傳
- 公開資料夾（`public/`）
- 線上編輯器
- 管理員儀表板

### 資料夾存取控制

```python
# 對每個檔案操作做檢查
def user_folder(username):
    return os.path.join(ROOT, username)

def allowed_path(path, username):
    user_root = user_folder(username)
    resolved = os.path.abspath(path)
    return resolved.startswith(os.path.abspath(user_root))
```

使用者無法存取 `/data/box5/alice/` 以外的任何資料。

### 與 Docker 模式共用功能

| 功能 | 多用戶（無容器） | 多用戶（Docker 隔離） |
|------|------------------|------------------------|
| Web 介面 | 有 | 有 |
| 帳號系統 | 有 | 有 |
| 公開資料夾 | 有 | 有 |
| 管理員功能 | 有 | 有 |
| API Key | 有 | 有 |
| 登入記錄 | 有 | 有 |
| 配額系統 | 有 | 有 |
| 線上編輯器 | 有 | 有 |
| SSH 存取 | 無 | 有 |
| 容器隔離 | 無 | 有 |
| 獨立環境 | 無 | 有 |

### 實作要點

- 移除所有 Docker 容器建立邏輯
- 所有檔案操作加上路徑檢查（不可超出 `<root>/<username>/`）
- `public/` 資料夾為公開，其他為私人
- 沿用現有 auth、admin、API 功能

## 模式三：多用戶模式（Docker 隔離）

現有的完整模式，不變動。

```bash
cd k8s
BOX5_MODE=docker        # 預設
BOX5_IMAGE=box5-server:latest
python -m uvicorn main:app --host 0.0.0.0 --port 8000
```

---

## 模式切換

### 環境變數

| 變數 | 說明 | 選項 |
|------|------|------|
| `BOX5_MODE` | 模式 | `simple` / `multi` / `docker` |
| `BOX5_ROOT` | 資料根目錄 | 路徑 |
| `BOX5_NO_DOCKER` | 停用 Docker | `1` = 停用 |

### 自動偵測

```python
MODE = os.getenv("BOX5_MODE", "docker")

if MODE == "simple":
    init_simple_mode()
elif MODE == "multi":
    init_multi_mode()
else:
    init_docker_mode()
```

### 啟動範例

```bash
# 簡單模式（個人使用）
BOX5_MODE=simple BOX5_ROOT=/data/box5 python -m uvicorn main:app --port 8000

# 多用戶（無容器，小團隊）
BOX5_MODE=multi BOX5_ROOT=/data/box5 python -m uvicorn main:app --port 8000

# 多用戶（Docker，生產環境）
BOX5_MODE=docker BOX5_IMAGE=box5-server:latest python -m uvicorn main:app --port 8000
```
