# box5-k8s v1.1

Multi-tenant box5 with isolated Docker containers per user.

## Features

- Each user gets their own Docker container running box5-server
- Users can register and login, container created automatically
- Home page (`/`) shows files from user's sync folder
- Editor (`/editor`) provides VSCode-like interface
- File viewing supports markdown, HTML, images, text
- User files stored in shared volume mounted to both container locations

## Architecture

- Main k8s server (port 8000) - serves web UI, manages containers
- User containers - each running box5-server (port 3111)
- Volume mount: host `uploads/{username}/sync` → container `/tmp/box5/sync`
- Database: SQLite with WAL mode for thread safety

## Quick Start

```bash
cd /Users/Shared/ccc/project/box5/k8s
./k8s_run.sh
```

Then visit:
- http://localhost:8000/ - File browser
- http://localhost:8000/editor - Web editor
- http://localhost:8000/login - Login/Register

Default user: username `ccc`, password `cccpass`

## Building Docker Image

```bash
docker build -t box5-server:latest -f Dockerfile.box5 .
```

## Testing

```bash
./test.sh
```

## File Path Resolution

- Home page reads from mounted volume: `uploads/{username}/sync/`
- Editor reads from container: `/tmp/box5/sync`
- Both locations are the same due to volume mount
- View endpoint serves files directly with proper Content-Type

## Environment Variables

- `DEFAULT_USER` - default user (default: ccc)
- `DEFAULT_PASS` - default password (default: cccpass)
- `BOX5_IMAGE` - Docker image name (default: box5-server:latest)
- `BOX5_HOST` - host for user servers (default: localhost)
